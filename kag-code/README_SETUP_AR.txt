============================================================
 مركز القيادة المباشر — حدائق الملك عبدالله
 نسخة Google Sheet (V60)
 البيانات تُعبّأ في Google Sheet وتُسحب وتتحدث تلقائيًا
============================================================

ماذا تغيّر في هذه النسخة؟
- مصدر البيانات أصبح Google Sheet واحد يعبّيه الفريق.
- الخادم يسحب البيانات من الجدول تلقائيًا، ويحدّثها لكل المستخدمين خلال ثوانٍ.
- تمت إزالة كلمة المرور نهائيًا من ملفات الواجهة (كانت ثغرة أمنية)، والتحقق الآن من الخادم فقط.
- تم اعتماد المسارات الخمسة بشكل كامل (أ، ب، ج، د، هـ).
- نسبة التقدم وحالة كل مسار تُحتسب آليًا من عناصر الجدول.
- التغذية الحية (Feed) أصبحت مشتقة من بيانات حقيقية بدل رسائل عشوائية.

------------------------------------------------------------
أولًا: المتطلبات
------------------------------------------------------------
- جهاز عليه Node.js إصدار 18 أو أحدث.

------------------------------------------------------------
ثانيًا: تجهيز Google Sheet
------------------------------------------------------------
1) أنشئ Google Sheet جديد.
2) في التبويب الأول، اجعل الصف الأول عناوين الأعمدة التالية بالترتيب:
      المسار | النوع | العنوان | المسؤول | الحالة | التاريخ
   (أسهل طريقة: في Google Sheets اختر File > Import وارفع الملف
    public/KAG_GoogleSheet_Template.csv المرفق، فيتعبّى الجدول جاهزًا.)
3) عبِّ الصفوف ببيانات المشروع.
4) شارك الجدول للقراءة:
      Share > General access > Anyone with the link > Viewer
   (هذا ضروري حتى يقرأ الخادم البيانات دون أي كلمات مرور.)
5) انسخ معرّف الجدول SHEET_ID من الرابط:
      https://docs.google.com/spreadsheets/d/(((SHEET_ID)))/edit

القيم المقبولة داخل الجدول:
- المسار:  أ / ب / ج / د   (أو A / B / C / D)
- النوع:   مهمة / مخاطرة / تصريح / معلم رئيسي
           (أو tasks / risks / permits / milestones)
- الحالة:  للإنجاز: مكتملة أو معتمدة
           قيد العمل: قيد التنفيذ أو تحت المتابعة
           للخطر: معرضة للخطر
- التاريخ: مثال 2026-09-12

ملاحظة: نسبة تقدم كل مسار = (عدد المهام المكتملة ÷ إجمالي مهام المسار).

------------------------------------------------------------
ثالثًا: التشغيل
------------------------------------------------------------
افتح مجلد المشروع في Terminal، ثم:

• على Mac / Linux:
    SHEET_ID=ضع_المعرّف_هنا ADMIN_PASSWORD=كلمة_مرور_قوية node server.js

• على Windows (PowerShell):
    $env:SHEET_ID="ضع_المعرّف_هنا"
    $env:ADMIN_PASSWORD="كلمة_مرور_قوية"
    node server.js

ثم افتح المتصفح على:
    http://localhost:3000

متغيرات اختيارية:
- SHEET_NAME       اسم التبويب إذا لم يكن الأول (مثال: "المهام").
- SHEET_REFRESH_MS مدة إعادة السحب من الجدول بالمللي ثانية (افتراضي 15000).
- OPENING_DATE     تاريخ الافتتاح (افتراضي 2026-09-27).
- ADMIN_USERNAME   اسم مستخدم الأدمن (افتراضي MAYADEEN).
- PORT             المنفذ (افتراضي 3000).

إذا لم تُحدِّد SHEET_ID، يعمل الموقع ببيانات تجريبية حتى تجهّز الجدول.
إذا لم تُحدِّد ADMIN_PASSWORD، يطبع الخادم كلمة مرور عشوائية في الـ Terminal عند الإقلاع.

------------------------------------------------------------
رابعًا: كيف يتحدّث كل شيء؟
------------------------------------------------------------
- عدّل أي بيانات في Google Sheet.
- الخادم يسحب التغييرات تلقائيًا (كل 15 ثانية افتراضيًا).
- متصفحات جميع المستخدمين تُحدّث الشاشة تلقائيًا (كل 3 ثوانٍ).
- لتحديث فوري: ادخل كأدمن > صفحة "مصدر البيانات" > زر "إعادة المزامنة الآن".

شريط الحالة أعلى الصفحة يبيّن حالة الاتصال وآخر تحديث للجميع.

------------------------------------------------------------
خامسًا: ملاحظات أمنية
------------------------------------------------------------
- لا توجد أي كلمة مرور داخل ملفات الواجهة بعد الآن.
- البيانات للقراءة فقط داخل اللوحة؛ التعديل يتم من Google Sheet مباشرة.
- تسجيل دخول الأدمن مطلوب فقط لإعدادات العرض/البث (وليس للبيانات).
- قراءة حالة اللوحة (GET /api/state) متاحة لمن يصل للرابط، لذا إن أردت
  تقييد الوصول بالكامل، شغّل الموقع داخل شبكتك الداخلية أو خلف VPN/بوابة.

------------------------------------------------------------
مرجع الأعمدة
------------------------------------------------------------
المسار   = حرف المسار (أ–هـ)
النوع    = مهمة / مخاطرة / تصريح / معلم رئيسي
العنوان  = اسم العنصر
المسؤول  = الجهة أو الشخص المسؤول
الحالة   = حالة العنصر (تُستخدم لاحتساب التقدم والمخاطر)
التاريخ  = تاريخ الاستحقاق (YYYY-MM-DD)

------------------------------------------------------------
تحديث مهم: تم تثبيت معرّف جدولك مسبقًا
------------------------------------------------------------
معرّف الجدول الخاص بك أصبح مثبّتًا داخل server.js، لذلك يكفي تشغيل:
    ADMIN_PASSWORD=كلمة_قوية node server.js
بدون الحاجة لكتابة SHEET_ID في كل مرة.

للتأكد أن الجدول مقروء (قبل التشغيل): افتح هذا الرابط في متصفحك
وأنت غير مسجّل دخول (وضع التصفح الخفي). يجب أن يُنزّل/يظهر ملف CSV ببياناتك:
    https://docs.google.com/spreadsheets/d/15m2VHVr7W2mWWz7g_Z5iMDxaIZuRshj-N3EtA9j4lWk/gviz/tq?tqx=out:csv
إذا ظهرت صفحة تسجيل دخول جوجل أو رسالة منع => المشاركة غير مضبوطة،
ارجع لخطوة المشاركة: Anyone with the link > Viewer.

------------------------------------------------------------
التحصين الأمني (نسخة V60-Secure)
------------------------------------------------------------
تم تطبيق طبقات حماية إضافية:
- منع XSS: تهريب كل بيانات الجدول من الخادم، بحيث لا يمكن تنفيذ أي كود
  حتى لو كُتب داخل خلية في Google Sheet.
- رؤوس أمان كاملة: Content-Security-Policy، منع التأطير (X-Frame-Options: DENY)،
  nosniff، Referrer-Policy، Permissions-Policy، وHSTS عند التشغيل على HTTPS.
- تسجيل الدخول: مقارنة آمنة زمنيًا + حظر مؤقت بعد عدة محاولات خاطئة
  (٦ محاولات خلال ١٥ دقيقة => حظر ١٥ دقيقة لكل عنوان IP).
- حماية CSRF: رفض طلبات تسجيل الدخول/الخروج/المزامنة من مصادر خارجية.
- صفحة «مصدر البيانات» وإعادة المزامنة أصبحت للأدمن فقط (لا تُسرّب رابط الجدول).
- لا تُعرض رسائل الأخطاء الداخلية للمستخدم العام.
- مهلة لقراءة الجدول + حماية ضد هجمات الإبطاء (slowloris).
- كوكي الجلسة: HttpOnly + SameSite=Lax + Secure (تلقائيًا على HTTPS).

قفل كامل اختياري:
  لإجبار تسجيل الدخول حتى لمشاهدة اللوحة، شغّل:
    REQUIRE_LOGIN=true ADMIN_PASSWORD=كلمة_قوية node server.js

توصيات للنشر على الإنترنت:
- شغّل خلف HTTPS (مثلاً عبر Nginx/Caddy) لتفعيل Secure وHSTS تلقائيًا.
- اضبط ADMIN_PASSWORD بكلمة قوية، ولا تستخدم الكلمة المولّدة في الإنتاج.
- يفضّل إبقاء اللوحة داخل الشبكة الداخلية أو خلف VPN إن كانت البيانات حساسة.

------------------------------------------------------------
حل مشكلة: «الوصول للشيت ما تم»
------------------------------------------------------------
السبب الأكثر شيوعًا: الجدول غير مُشارَك للعموم (وضع Restricted).
عند فحص جدولك رجع 401 (غير مصرّح) = ليس عامًا بعد.

الحل (الأضمن) — النشر للويب:
1) في Google Sheets: File ← Share ← Publish to web
2) اختر التبويب المطلوب، والصيغة: Comma-separated values (.csv)
3) اضغط Publish وانسخ الرابط (يبدأ بـ /spreadsheets/d/e/... وينتهي بـ output=csv)
4) شغّل الخادم بهذا الرابط:
     SHEET_CSV_URL="الصق_رابط_النشر_هنا" ADMIN_PASSWORD=كلمة_قوية node server.js

الحل البديل — مشاركة عادية:
1) Share ← General access ← Anyone with the link ← Viewer ← Done
2) شغّل: node server.js  (المعرّف مثبّت مسبقًا)

تأكيد سريع (في وضع التصفح الخفي):
افتح هذا الرابط؛ يجب أن ينزّل ملف CSV ببياناتك، لا صفحة تسجيل دخول:
   https://docs.google.com/spreadsheets/d/15m2VHVr7W2mWWz7g_Z5iMDxaIZuRshj-N3EtA9j4lWk/export?format=csv

حل احتياطي تام (بدون إنترنت/مشاركة):
نزّل الجدول كـ CSV (File ← Download ← CSV) وشغّل:
   LOCAL_CSV="/المسار/الكامل/للملف.csv" node server.js

ملاحظة: الخادم الآن يجرّب تلقائيًا أكثر من طريقة قراءة (gviz ثم export)،
ويعطي رسالة واضحة في صفحة «مصدر البيانات» إن كان الجدول غير عام.
